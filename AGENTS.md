# Repository Instructions

- 各 Codex セッションの終了時は、次セッション向けに `HANDOFF.md` を確認・更新する。
- 引き継ぎ書はリポジトリ直下の `HANDOFF.md` 1枚に集約し、別ファイルを増やさない。
- `HANDOFF.md` は `全体の概要`、`現在の作業段階`、`次のステップの作業` を基本構成にする。
- 解決済み・実施済みの詳細は短く要約し、未解決の問題、再開手順、検証コマンド、実際に使ったサーバ起動 option を優先して残す。
- 実際には確認していないことを確認済みとして書かない。未実施の WebUI / LLM / TTS 検証は明示する。
